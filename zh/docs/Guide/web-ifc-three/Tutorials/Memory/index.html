简介#

📭📫 在很多情况下，我们会希望创建能够打开和关闭 BIM 浏览器的应用程序。虽然这看起来很明显，但有一个问题。内存管理。

🤔 现在很多人会挠头，因为内存管理在网络应用中不是一个常见的问题。JavaScript 和其他现代语言都有自动内存管理/垃圾回收器，所以程序员不必担心释放对象和数组的问题。

✍🤖❓ 然而，在使用 Three.js 时，前往显卡的数据（如缓冲区）不受这种自动内存管理的影响。

🤕 这意味着，如果我们关闭一个用 Three.js 制作的浏览器（无论是否有 IFC.js），我们会产生一个内存泄漏的现象。也就是说，我们阻断了用户的一部分 RAM 内存。如果泄漏太大，应用程序将变得越来越慢，并可能崩溃。

💡
你可以在这里找到关于这个主题的更多信息。

🚀 如果我们正在创建一个SPA（单页应用），例如使用React或Angular等框架和库，这就变得尤为关键。在这些情况下，网络应用程序永远不会被重新加载，而内存泄漏就会累积起来。

幸运的是，IFC.js 已经考虑到了这一点，并允许你非常容易地**释放使用的内存。让我们来看看如何。👇

你可以找到这个教程的完整例子这里。

如何做到这一点#

监控内存#

👀 如果你没怎么用过 Three.js 的优化场景，你的第一个问题可能是如何查看你的应用程序消耗了多少内存。否则，就不可能发现内存泄漏。

🌍 有几种方法可以做到这一点。首先是使用你正在使用的浏览器的开发工具。例如，在谷歌浏览器中，有一个名为 "内存 "的部分，你可以进行快照，看看当前标签页消耗了多少内存。

🤖 不过，更方便的方法是使用库stats.js。这个库可以监测 Three.js 应用程序的性能和内存。它可以用npm i stats.js来安装，使用方法如下。

// 统计数据
const stats = new Stats();
stats.showPanel(2);
document.body.append(stats.dom);
// 动画循环
const animate = () => {
stats.begin();
// 更新其他内容
stats.end();
requestAnimationFrame(animate);
};
animate();

📋 这将带出执行期间的场景统计。通过点击它们，我们可以在应用程序的性能和内存使用之间进行切换。

处置内存#

✌ 现在让我们创建一个函数，删除 IFC.js 所消耗的所有内存。有两件事需要注意。

  • 如果我们之前指定了WASM 文件的路径，我们将不得不在 web-ifc-three 的新实例中再次这样做。

  • 这个很重要：如果我们在数组、对象或类中存储了对加载的 IFC 模型的引用，我们必须手动删除它们。否则，内存将不会被释放。

async function releaseMemory() {
// 这将释放所有的IFCLoader内存
await ifcLoader.ifcManager.dispose();
ifcLoader = null;
ifcLoader = new IFCLoader();
// 如果之前设置了wasm路径，
// 我们需要重置它
await ifcLoader.ifcManager.setWasmPath("../../../");
// 如果IFC模型是一个数组或对象，
// 你也必须在那里释放它们。
// 否则，它们将不会被垃圾回收。
models.length = 0;
}

设置简单的 GUI#

💅 最后，我们要创建一个 HTML 按钮，并将其链接到我们先前创建的函数。

<input type="button" id="memory-button" value="Release memory" />
// 设置内存处理
const button = document.getElementById("memory-button");
button.addEventListener(`click`, () => releaseMemory());

😎 这就是应用程序的样子。

尝试加载一个模型并释放内存：你会看到它恢复正常。请注意，它可能需要几秒钟，因为浏览器的垃圾收集器需要一些时间。另外，你也可以使用浏览器的开发工具手动应用垃圾收集器。

SPA#

🌈 创建SPA或单页应用程序是很常见的，因为它们更有效率，需要与后台进行更少的沟通。制作这些应用程序的工具的例子有React、Angular、Vue等。

👶👴 在许多情况下，这些工具会自动管理应用程序组件的生命周期。例如，当用户关闭 3D 视图时，包含该视图的 HTML 元素会自动销毁。

🚧🚧🚧 为了避免内存泄漏，有必要在每次销毁包含 3D 视图的组件时本教程中看到的释放应用程序内存。在许多情况下，有钩子或类似的机制在每次组件被销毁时自动执行这一逻辑。
