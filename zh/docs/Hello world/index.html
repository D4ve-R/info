你好，世界

简介#

⚡ 用 IFC.js 创建一个 BIM 应用程序是非常容易的。你可以找到指南的完整项目这里和部署的应用程序这里。你只需要有Node.js和任何 IDE 的安装，就可以按照下面的步骤进行。另外，如果你没有任何 IFC 文件来跟随，你可以得到一些这里。

{" "}

💡
使用IFC.js需要有网络开发（HTML、CSS、JavaScript）和Three.js的基本知识。如果您以前没有使用Three.js的经验，您也许应该看看这里。

🐣 这个最小的教程将用 Vanilla JavaScript 完成，不使用像 React、Vue、Angular、Svelte 等框架。然而，同样的步骤可以适应并应用于任何这些用例。

设置项目#

安装图书馆#

👩‍🏫 首先要做的是创建一个空文件夹，用npm init命令启动一个新的 npm 项目。这将生成一个package.json文件，其中包含一些数据，如项目名称、版本、命令和依赖性。此外，以下依赖项必须与 npm 一起安装。

// 安装IFC.js
npm i web-ifc-three
// 安装Three.js
npm i three
// 安装一个捆绑程序：我们将在本指南中使用rollup.js。
npm i rollup --save-dev
npm i @rollup/plugin-node-resolve --save-dev

📝 下一步是创建一个名为index.html的 HTML 文件作为应用程序的主文件。该 HTML 将有。

  • 一个canvas 元素，用于渲染 Three.js 场景。
  • 一个输入元素，它将从我们的计算机上打开 IFC 文件到应用程序中。
  • 一个脚本，引用一个叫bundle.js的文件，这是我们将用 rollup 制作的应用程序的捆绑文件。
<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8" />
<meta http-equiv="X-UA-Compatible" content="IE=edge" />
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
<link rel="stylesheet" href="styles.css" />
<title>Document</title>
</head>
<body>
<input type="file" name="load" id="file-input" />
<canvas id="three-canvas"></canvas>
<script src="bundle.js"></script>
</body>
</html>

添加样式#

🎨 下面的 CSS 文件将使画布成为全屏。

* {
margin: 0;
padding: 0;
box-sizing: border-box;
}
html,
body {
overflow: hidden;
}
#three-canvas {
position: fixed;
top: 0;
left: 0;
outline: none;
}
#file-input {
z-index: 1;
position: absolute;
}

代码捆绑#

接下来，我们将创建 rollup 配置文件。这个文件必须称为rollup.config.js，包括对我们之前安装的插件的引用。

{" "}

🧻

Rollup?

Rollup是一个非常流行的捆绑库。例如，它是 Three.js 使用的捆绑器。请看一下rollup docs，以了解更多。

import resolve from "@rollup/plugin-node-resolve";
export default {
input: "src/app.js",
output: [
{
format: "esm",
file: "src/bundle.js",
},
],
plugins: [resolve()],
};

🎯 另外，package.json文件需要被修改，以包含控制 rollup 的命令。在每个命令中，你必须指定你的 rollup 配置文件的相对路径。如果我们已经正确安装了一切，我们应该在这个文件中看到相同的依赖关系（库的版本可能不同）。

  • npm run build将捆绑项目并在项目的根目录下创建一个名为bundle.js的文件。

  • npm run watch将激活watch模式，在我们每次修改代码并保存时自动更新该文件。

{
"name": "example",
"version": "1.0.0",
"description": "-",
"main": "app.js",
"scripts": {
"build": "rollup -c ./rollup.config.js",
"watch": "rollup -w -c ./rollup.config.js"
},
"author": "",
"license": "ISC",
"devDependencies": {
"@rollup/plugin-node-resolve": "^11.2.1",
"rollup": "^2.45.2"
},
"dependencies": {
"three": "^0.128.0",
"web-ifc-three": "0.0.102"
}
}

WebAssembly#

📁 接下来要做的是把web-ifc.wasm和web-ifc-mt.wasm文件复制到你项目中的一个目录。它可以在node_modules\web-ifc中找到（如果你只使用 Three 的 IFCLoader，则可以在node_modules\three\examples\jsmloaders\ifc中找到）。我们可以把它们复制到我们想要的地方；在这个例子中，它们将被复制到项目根目录下一个叫 wasm 的文件夹中。

这些文件是必要的，因为它们包含了web-ifc的编译的 C++逻辑，它是以本地速度读写 IFC 文件的解析核心。

💡
这些文件必须在你的应用程序中静态地提供。如果你使用的是React、Angular、Vue或Svelte等框架或库，这可能需要进行不同的调整。

设置一个 3D 场景#

最后，我们要创建一个 JavaScript 文件，为我们的应用程序编写代码。这个文件可以位于任何地方，有任何名字，但你必须在rollup.config.js中反映出来。

🧱 我们将使用 Three.js 创建一个基本的 3D 场景。

import {
AmbientLight,
AxesHelper,
DirectionalLight,
GridHelper,
PerspectiveCamera,
Scene,
WebGLRenderer,
} from "three";
import { OrbitControls } from "three/examples/jsm/controls/OrbitControls";
// 创建Three.js场景
const scene = new Scene();
//Object to store the size of the viewport
const size = {
width: window.innerWidth,
height: window.innerHeight,
};
// 创建摄像机（用户的视角）。
const aspect = size.width / size.height;
const camera = new PerspectiveCamera(75, aspect);
camera.position.z = 15;
camera.position.y = 13;
camera.position.x = 8;
// 创建场景的灯光
const lightColor = 0xffffff;
const ambientLight = new AmbientLight(lightColor, 0.5);
scene.add(ambientLight);
const directionalLight = new DirectionalLight(lightColor, 1);
directionalLight.position.set(0, 10, 0);
directionalLight.target.position.set(-5, 0, 0);
scene.add(directionalLight);
scene.add(directionalLight.target);
// 设置渲染器，获取HTML的画布。
const threeCanvas = document.getElementById("three-canvas");
const renderer = new WebGLRenderer({
canvas: threeCanvas,
alpha: true,
});
renderer.setSize(size.width, size.height);
renderer.setPixelRatio(Math.min(window.devicePixelRatio, 2));
// 在场景中创建网格和坐标轴
const grid = new GridHelper(50, 30);
scene.add(grid);
const axes = new AxesHelper();
axes.material.depthTest = false;
axes.renderOrder = 1;
scene.add(axes);
// 创建轨道控制（用于导航场景）。
const controls = new OrbitControls(camera, threeCanvas);
controls.enableDamping = true;
controls.target.set(-2, 0, 0);
// 动画循环
const animate = () => {
controls.update();
renderer.render(scene, camera);
requestAnimationFrame(animate);
};
animate();
// 根据浏览器的大小调整视口
window.addEventListener("resize", () => {
size.width = window.innerWidth;
size.height = window.innerHeight;
camera.aspect = size.width / size.height;
camera.updateProjectionMatrix();
renderer.setSize(size.width, size.height);
});

🌏 要在本地运行该应用程序，我们需要一个本地服务器。如果你使用 VS Code 作为 IDE，一个选择是安装实时服务器扩展，它允许我们打开谷歌浏览器的一个实例，运行我们的网络应用程序，并实时查看我们对代码的修改。

加载 IFC 文件#

加载用户的模型#

🏠 最后，我们将使用 IFC.js 来加载 IFC 文件。这可以通过实例化加载器并为用户向 HTML 输入元素上传 IFC 文件时创建一个事件来完成。

import { IFCLoader } from "web-ifc-three/IFCLoader";
// 设置IFC加载
const ifcLoader = new IFCLoader();
const input = document.getElementById("file-input");
input.addEventListener(
"change",
(changed) => {
const file = changed.target.files[0];
var ifcURL = URL.createObjectURL(file);
ifcLoader.load(ifcURL, (ifcModel) => scene.add(ifcModel));
},
false
);

🎣 请记住，如果你没有把 wasm 文件保存在项目的服务文件根目录下，你需要用setWasmPath指定它的位置。例如，如果我们把它们保存在项目根目录下的wasm文件夹中，该文件夹名为static，我们将做如下工作。

ifcLoader.ifcManager.setWasmPath("static/wasm/");

如果你所做的一切都正确，你应该能够在你的本地服务器中看到与这类似的东西。从这里开始，可能性是无穷的。

加载我们的模型#

🤔 在上一点中，我们看到如何直接加载 BIM 模型，这很好。如果我们想显示我们的 BIM 模型而不是让用户上传他们的模型呢？这很简单。一般来说，有两种可能性。

  • 把 IFC 放在你想显示它的同一个应用程序里。

  • 必须从外部存储服务中获取它。

    🎩 在第一种情况下，只需引用 IFC 文件的 URL 即可。就是说，它在应用程序中的相对路径。例如，如果 IFC 在项目根部一个叫 "models "的文件夹中，我们可以在启动应用程序时加载该 IFC，如下所示。

ifcLoader.load("models/Example_model.ifc", (ifcModel) => scene.add(ifcModel));
💡
从远程存储服务中获取文件的方法因使用的服务而异。然而，逻辑是相同的：获取信息，创建一个URL并将其作为参数传递给IFCLoader。
