📃 プロパティ

はじめに#

🧬🧬🧬️ ほぼすべての BIM モデルにはジオメトリが含まれていますが、本当に重要なのは BIM の「I」、つまり情報であることは誰もが認めるところでしょう。IFC.js は、どんな IFC ファイルでも上から下まで、ネイティブなスピードで、ほとんどコードなしにトラバースすることができます。

💾 これにより、クライアント側で IFC**ファイルから直接情報を読み取るウェブアプリケーションを作成できるだけでなく、その情報をリレーショナルまたはノンリレーショナルデータベースに転送し、バックエンドで処理することも可能になりました。

💡
IFC.jsは、IFCファイルをID別、空間ツリー別、フィルタリング検索で読み取ることができます。

📚 IFC スキームにはいくつかのタイプのプロパティがあり、それぞれが特定の目的を持っていますが、IFC.js はそれらすべてを 取得 できます。最も一般的なものをいくつか紹介します。

📕

ネイティブ・プロパティ

IFCの各クラスに固有のもの。

📗

タイプのプロパティ

同じタイプのすべての要素の特性を記述する（例：特定のタイプのすべてのエンベロープの壁）。

📘

材料の特性

その要素のレイヤーを構成するすべての素材を記述します。

📙

プロパティセット

ユーザー定義のプロパティの任意のセット。1つまたは複数の要素に関連する複数のプロパティセットが存在する可能性があります。各プロパティセットには、相互に関連する任意のプロパティ群が含まれる。

📒

量の特性

参照している要素の寸法を記述するプロパティのセット。ジオメトリの定義から暗黙的に要素の寸法を推測することも可能ですが、このように明示的に記述することで、IFCモデルを計測するアプリケーションの作成が非常に容易になります。

🚴‍♂️ しかし、理論はもういい。早速、作業に取り掛かりましょう。

その方法とは#

ハロープロパティ#

👩‍🏫 IFCからプロパティを抽出する基本的な方法は、要素のIDから抽出することです。多くの場合、ユーザーが要素を選択したときに、その要素の直接・間接のプロパティを取得したいと考えます。これまでのチュートリアルでは、そのIDを簡単に取得する方法を紹介してきました。

☝ さて、ID がわかったところで、要素のプロパティを取得するのは難しいでしょうか？ アイテム選択のチュートリアル で見たものに1 行のコードを追加するだけで、ダブルクリックしたときに選択した要素のプロパティを見ることができます。

// アイテムがピックされたときに実行されるイベント
async function pick(event) {
const found = cast(event)[0];
if (found) {
const index = found.faceIndex;
const geometry = found.object.geometry;
const ifc = ifcLoader.ifcManager;
const id = ifc.getExpressId(geometry, index);
const modelID = found.object.modelID;
const props = await ifc.getItemProperties(modelID, id);
output.innerHTML = JSON.stringify(props, null, 2);
}
}

🤨 しかし、その特性とは何でしょうか？

これらはネイティブ・プロパティ、つまり、各 ifc 要素タイプに固有の基本的なプロパティです。プロパティはJavaScript オブジェクトとして与えられるので、非常に簡単にトラバースすることができます。

🤔 数値を値として持つプロパティがあるのはなぜですか？

これは数字ではなく、express IDです。つまり、その ID を使って再びgetItemProperties()を使えば、その参照された要素のプロパティが得られるということです。

これがIFCの仕組みです。各 要素はプロパティを持ち、各プロパティは他の要素への参照となります。そのため、基本的には参照を介してIFC全体を横断することができます。

😑 リファレンス？しかし、私はすべての情報を直接欲しい！。

心配はご無用です。オプションでgetItemProperties()の 3 番目のパラメータを使うことができます。これはブール値で、デフォルトでは false です。これを true に設定すると、参照されているすべての要素を再帰的に取得し、すべてを 1 つの JavaScript オブジェクトにうまくまとめて表示します。

とはいえ、ランタイムのオーバーヘッドを避けるために、通常はこれを避けることをお勧めします。必要なときに必要な ID だけにgetItemProperties()を使用する方がはるかに良いでしょう。

🙄 他のタイプのプロパティを取得するにはどうすればいいですか？

IFC.js の API には、他のすべてのプロパティタイプに対して同様のメソッドがあります。すなわち、getTypeProperties()、getMaterialProperties()、getPropertySets()です。この最後のメソッドは、プロパティセットと数量セットの両方を取得します。

😨 IFC を横断するには、getItemProperties()しかないのでしょうか？

そんなことはありません。次は、空間構造全体を横断する方法を見てみましょう。

IFC を使いこなす#

🔥 要素の ID から IFC 情報を参照できるようになりましたし、シーン内のオブジェクトをクリックして ID を取得する方法も知っています。しかし、これだけでは不十分なのです。

  • 幾何学的な表現を持たない要素（例：IfcSite）のプロパティを取得したい場合は？

  • IFC の空間構造全体を横断したい場合は？

🌈 もちろん、IFC.js の API を使えば、これらのことはとても簡単です。

空間ツリー#

🌳🌲🌴 ご存知のように、すべての IFC には、空間要素構造（ここでは空間ツリーと呼ぶ）と呼ばれる一般的な構造があります。これは、プロジェクト内のすべての製品（物理的オブジェクト）の相対的な位置を定義する要素で構成されています。

💡
空間構造は一般的に、IfcProject、IfcSite、IfcBuilding、IfcBuildingStorey、IfcSpaceで構成されています。

プロジェクトの空間構造を取得するには、単純に getSpatialStructure() を呼び出します。ここで、唯一のパラメータは modelID です。このメソッドはプロジェクトの空間構造全体を JavaScript のオブジェクトとして返します。

🚧 これは、プロジェクトのすべての製品のすべてのプロパティを取得することを意味しますか？いいえ、それは計算量が多くなるからです。その代わり、次のような情報を持つアイテムのツリーが得られます。

interface Node {
expressID: number;
type: string;
children: Node[];
}

👩‍🏫 type（例：IfcWall）とexpress IDしか取得できないことに注意してください。要素のプロパティを取得したい場合は、ツリーをトラバースして、すべての ID に対して上述のプロパティゲッターメソッドを呼び出す必要があります。

ユーザーにプロパティを表示することが目的であれば、一般的には、ユーザーが要求したとき（例：GUIで特定のアイテムをクリックしたとき）にのみプロパティを取得する方が良いでしょう。

例えば、プロジェクトの空間構造を要求すると、次のようなものが出てきます。

{
expressID: 100,
type: "IfcProject",
children: [{
expressID: 101,
type: "IfcSite",
children: [{
expressID: 102,
type: "IfcBuilding",
children:[
{
expressID: 103,
type: "IfcBuildingStorey",
children: {...}
},
{
expressID: 104,
type: "IfcBuildingStorey",
children: {...}
}
]
}]
}]
}

タイプ別にアイテムを取得#

👩‍🏫 特定のタイプのすべての要素を取得したい場合があります（例：プロジェクト内のすべてのIfcWallインスタンス）。このような場合、要素をクリックして選択したり、プロジェクトの空間ツリーを取得することは有用ではありません。

🎸🪕🎻 そのためにあるのが getAllItemsOfType メソッドです。使い方はとても簡単で、web-ifcから直接タイプをインポートすることができます。例えば、プロジェクトの全てのIfcSlabインスタンスのプロパティを取得するには、以下のようにします。

import { IFCSLAB } from "web-ifc";
const ifc = ifcLoader.ifcManager;
const modelID = 0;
async function logAllSlabs() {
const slabsID = await ifc.getAllItemsOfType(modelID, IFCSLAB);
for (let i = 0; i <= slabsID.length; i++) {
const slabID = slabsID[i];
const slabProperties = await ifc.getItemProperties(0, slabID);
console.log(slabProperties);
}
}
logAllSlabs();
