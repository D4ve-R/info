API

web-ifc-three でできるすごいことリスト#

📃 すべての API はドキュメント化されていますので、このドキュメントで紹介されているオブジェクトやメソッドを使用すると、使用している IDE にかかわらず、Intellisense でヘルプが表示されるはずです。チェックしてみてください！

👩‍🏫 しかし、インテリセンスやコメントを読むことは、最も快適なことではないことを理解しているので、このページでは、API でできることの概要を説明します。すべての内容は、後ほど個別のチュートリアルで詳しく説明します。

IfcLoader#

💎 ライブラリーからインポートする唯一のオブジェクトです。IFC を扱うために必要なすべてのロジックが含まれています。そのload()メソッドとloadAsync()メソッドを使用して、URL から IFC を読み込むことができます。他の Three.js Loader と同様です。例えば、IFC を読み込むには以下のようにします。

import { IFCLoader } from "web-ifc-three/IFCLoader";
const ifcLoader = new IFCLoader();
ifcLoader.load("models/Example_model.ifc", (ifcModel) => scene.add(ifcModel));

🏠🏠🏠 web-ifc-threeでは、シーン内に複数のモデルを読み込むことができます。API の操作の多くは、指定されたモデルに対して実行されます。どのモデルを操作したいかを表すには、そのモデルのModelIDを指定する必要があります。

🔍 モデルの ID は、生成されたメッシュのプロパティmodelIDで取得できます。

const modelID = ifcModel.modelID;

✌ この API にアクセスするには 2 つの方法があります。

  • IfcLoaderがloadとloadAsyncで生成するIfcModelのインスタンスを介して。

  • IfcLoader.IfcManagerを介して。

💡
IfcModelのインスタンスとIfcLoader.IfcManagerの両方は、APIにアクセスするために互換的に使用することができます。IfcModelを使用する場合、ModelIDを引数として与える必要はありません(暗黙的に知られています)。

🧰 セットアップ#

setWasmPath#

async IfcLoader.IfcManager.setWasmPath (
path: string
): void;

🧠 web-ifc.wasmとweb-ifc-mt.wasmのファイルの場所を指定します。これらのファイルは、web-ifcファイルであり、IFC.jsを使ってアプリケーションを作成する際に必要となります。これらのファイルは node_modules/web-ifc/` で見つけることができます。

🚧

使う道具には気をつけよう

React、Angular、Vue、Svelteなどのフレームワークやライブラリを使用している場合、プロジェクトのルートパスと提供されるアプリケーションのルートパスが一致しないことがあります。静的に提供されるファイルのパスがどのように管理されているか、各ケースで確認する必要があります。

関数の引数#

  • path web-ifc.wasmのルートです。

例#

web-ifc.wasm が dist/wasmDir にある場合。

await ifcLoader.setWasmPath("dist/wasmDir/");

setupThreeMeshBVH#

IfcLoader.IfcManager.setupThreeMeshBVH (
computeBoundsTree: any,
disposeBoundsTree: any,
acceleratedRaycast: any
): void;

⚡⚡⚡ この方法により、オブジェクトのピッキングが格段に速くなり、特に重いジオメトリを持つ非常に大きなモデルの場合に有効です。この方法により、IFC.js は Garrett Johnson 氏の素晴らしいライブラリを使用することができます。npm i three-mesh-bvhまたは yarn add three-mesh-bvh でインストールできます。

この方法は必須ではありませんが、中/大IFCモデルのオブジェクトを60fpsで選択できるようにしたい場合は必要です。

関数の引数#

  • computeBoundsTree 新しい BVH を構築し、それを boundsTree に割り当て、新しいインデックスバッファをジオメトリに適用する、プリメイドの BufferGeometry 拡張関数です。- ソース。

  • disposeBoundsTree BVH を破棄する BufferGeometry 拡張関数です。- ソースを参照してください。

  • acceleratedRaycast THREE.Mesh.raycast と同じシグネチャーを持つ、高速化されたレイキャスト関数です。BVH が利用可能な場合はレイキャスティングに使用し、そうでない場合はビルトインのアプローチに戻ります。- ソースを参照してください。

例#

import { IFCLoader } from "web-ifc-three/dist/IFCLoader";
import {
acceleratedRaycast,
computeBoundsTree,
disposeBoundsTree,
} from "three-mesh-bvh";
const ifcLoader = new IFCLoader();
ifcLoader.ifcManager.setupThreeMeshBVH(
acceleratedRaycast,
computeBoundsTree,
disposeBoundsTree
);

setOnProgress#

IfcLoader.IfcManager.setOnProgress (
onProgress: (event: ParserProgress) => void
): void;

⏳ IFC が 10％ロードされるごとに呼び出されるコールバック関数を設定します。これにより、ローディングバーを表示して、ユーザーにローディングの状況を示すことができます。

関数の引数#

  • onProgress: 10%ごとに呼び出されるコールバック関数です。この関数は、2 つのプロパティを持つオブジェクトを受け取る 1 つの引数を持つ必要があります。loaded（読み込まれたオブジェクトの数）と total（ファイル内のオブジェクトの数）です。

例#

function exampleCallback(event) {
const progress = (event.total / event.progress) * 100;
console.log("Progress: ", progress, "%");
}
ifcLoader.ifcManager.setOnProgress(exampleCallback);

applyWebIfcConfig#

async IfcLoader.IfcManager.applyWebIfcConfig (
settings: LoaderSettings
): void;

🎹🎹🎹 このライブラリが内部で使用しているパーサーであるweb-ifcの設定を適用します。

関数の引数#

  • settings: 以下のフィールドを含むオブジェクトです。

  • COORDINATE_TO_ORIGIN: boolean: モデルをシーンの中央に配置するかどうか。 ジオロケーションモデルに有効です。

  • USE_FAST_BOOLS: boolean: より高速な（正確さに欠ける）ブーリアンロジックを使用するかどうか。

  • BOOL_ABORT_THRESHOLD?: number: ブーリアン演算の計算の限界。

  • CIRCLE_SEGMENTS_LOW?: number: 低いセグメントの曲線の解像度。

  • CIRCLE_SEGMENTS_MEDIUM?: number: 中程度のセグメントの曲線（例：IfcSweptDiskSolid）の解像度です。

  • CIRCLE_SEGMENTS_HIGH?: number: 高セグメントの曲線（例：IfcCircle）の解像度です。

例#

ファイルが地理的に配置されていて、それをシーンの原点に持ってきたい場合。

await ifcLoader.ifcManager.applyWebIfcConfig({
COORDINATE_TO_ORIGIN: true,
USE_FAST_BOOLS: false,
});

useWebworkers#

async IfcLoader.IfcManager.useWebWorkers (
active: boolean,
path?: string
): void;

👷‍♀️👷‍♀️👷‍♀️ 解析ロジックを含むウェブワーカーをトグルします。API の残りの部分は同じですが、すべてのロジックはマシンの別のスレッドで実行されるため、重い操作でアプリがブロックされることはありません。

🚧 ファイルIFCWorker.jsをプロジェクトにコピーして、第 2 引数に相対パスを渡す必要があります。

関数の引数#

  • active: ウェブワーカーを使用するかどうか。
  • path: ワーカーファイルの相対パスです。active = trueの場合に必要です。

例#

というフォルダにIFCWorker.jsがある場合。files という名前のフォルダにあります。

await ifcLoader.ifcManager.useWebWorkers({
true,
"files/IFCWorker.js"
});

useJSONData#

async IfcLoader.IfcManager.useJSONData (
useJSON: boolean
): void;

🚀 WASM データの代わりに JSON プロパティデータを使用し、メモリ消費量が大幅に減少します。以下のシナリオでのみ使用してください。

  • IFC のプロパティにアクセスする必要がない場合は。
  • プロパティを JSON で提供する場合。 IFC ファイルからの JSON の取得は、以下のように事前に行うことができますこの例。

関数の引数#

  • useJSON: JSON データを使用するかどうか。

例#

await ifcLoader.ifcManager.useJSONData(true);

addModelJSONData#

async IfcLoader.IfcManager.addModelJSONData (
modelID: number,
data: { [id: number]: JSONObject
): void;

📦📦📦 モデルのプロパティを JSON データとして追加します。Web Worker を使用している場合は、オーバーヘッドを避けるために、代わりにloadJsonDataFromWorker()を使用してください。

関数の引数#

  • modelID: IFC モデルの ID です。
  • data: のデータを，キーが expressID，値がプロパティであるオブジェクトとして取得します。さらに、各値はプロパティとして、expressID（アイテムの ID）とtype（アイテムの IFC クラス）を持っています。

例#

await ifcLoader.ifcManager.addModelJSONData(0, jsonData);

loadJsonDataFromWorker#

async IfcLoader.IfcManager.loadJsonDataFromWorker (
modelID: number,
path: string
): void;

📦📦📦 JSON ファイルから IFC モデルのデータを Web Worker から直接読み込みます。ウェブワーカーを使用していない場合は、代わりに addModelJSONData() を使用してください。

関数の引数#

  • modelID: IFC モデルの ID です。
  • path: JSON ファイルへのパスは、Web Worker ファイルへの相対パスです。

例#

await ifcLoader.ifcManager.loadJsonDataFromWorker(0, "path/to/data.json");

🔍 ゲッターズ#

getExpressId#

IfcLoader.IfcManager.getExpressId (
geometry: BufferGeometry,
faceIndex: number
): number;

💳 IFC 要素の Express ID をフェースインデックスから取得する。

📜

なぜ顔認証なのか？

なぜなら、3D空間のオブジェクトをマウスで選択すると、その面のインデックスが得られるからです。通常、その面をハイライトしたり、分離したり、すべてのプロパティを取得するためには、その面が属するオブジェクトのIDが必要です。

関数の引数#

  • geometry マウスでクリックしたモデルの ジオメトリ です。

  • faceIndex raycasterで交差した顔のインデックスです。レイキャスターに慣れていない方もご安心ください。ピッキングについてのチュートリアルで詳しく説明します。

例#

const intersected = raycaster.intersectObject(mesh)[0];
const index = intersected.faceIndex;
const id = ifcLoader.ifcManager.getExpressId(mesh, index);

getIfcType#

IfcLoader.IfcManager.getIfcType (
modelID: number,
id: number,
): string;

🎸🪕🎻 指定された要素の IFC タイプを取得します（例：IFCWALL）。

💡
IFCの要素には、必ず関連するタイプがあります。IfcWall、IfcSlab、IfcWindow、IfcDoorなどです。

関数の引数#

  • modelID IFC モデルの ID です。

  • id プロパティを取得したいアイテムのエクスプレス ID。これは、getExpressId (3D ビューでオブジェクトを選択している場合)、または getAllItemsOfType や getSpatialStructure でモデルをトラバースすることで取得できます。

例#

const model = ifcModel.modelID;
const id = 2142;
const manager = loader.ifcLoader.ifcManager;
const type = manager.getIfcType(model, id);

getAllItemsOfType#

async IfcLoader.IfcManager.getAllItemsOfType (
modelID: number,
type: number,
verbose: boolean
): number[] | object[];

🎸🎸🎸 指定されたモデルの指定された IFC タイプのすべてのオブジェクト（すべての壁、すべての床、すべての窓など）を返します。これは、expressIDの配列を返すか、（verbose = trueの場合）見つかったアイテムのプロパティを含むオブジェクトの配列を返すことができます。

関数の引数#

  • modelID IFC モデルの ID です。

  • type 取得したい要素の IFC タイプです。これらのタイプは web-ifc から直接インポートすることができます (以下の例を参照)。

  • verbose true の場合、見つかったすべてのアイテムのプロパティを取得します。大きなモデルでは、この処理に時間がかかることがあるので、注意が必要です。

例#

import { IFCWALLSTANDARDCASE as W } from "web-ifc";
const manager = loader.ifcLoader.ifcManager;
const walls = await manager.getAllItemsOfType(0, W, false);

getItemProperties#

async IfcLoader.IfcManager.getItemProperties (
modelID: number,
id: number,
recursive = false
): object[];

📕🔍 与えられた要素のネイティブ・プロパティを取得します。IFC スキーマでは、プロパティには直接プロパティと間接プロパティの 2 種類があります。間接的なプロパティ（psets、qsets、type プロパティ）の取得方法については後述します。

💡
プロパティに関連するすべてのメソッドは、キーがプロパティの名前で、値がその値であるオブジェクトの配列を返します。

関数の引数#

  • modelID IFC モデルの ID です。

  • id プロパティを取得したいアイテムのエクスプレス ID。これは、getExpressId (3D ビューでオブジェクトを選択している場合)、または getAllItemsOfType や getSpatialStructure でモデルをトラバースすることで取得できます。

  • recursive true の場合、参照されているすべての要素のプロパティを再帰的に取得します。大きなモデルでは、この処理に時間がかかることがありますので、注意してください。

例#

const model = ifcModel.modelID;
const id = 2142;
const manager = loader.ifcLoader.ifcManager;
const props = await manager.getItemProperties(model, id, false);

getPropertySets#

async IfcLoader.IfcManager.getPropertySets (
modelID: number,
id: number,
recursive = false

📗🔍 与えられた要素のプロパティセットと数量セットを取得します。

Property sets?

例#

const model = ifcModel.modelID;
const id = 2142;
const manager = loader.ifcLoader.ifcManager;
const props = await manager.getPropertySets(model, id, false);

🧱 Subsets#

getSubset#

IfcLoader.IfcManager.getSubset (
modelID: number,
material?: Material
): object;

🧱🔍 指定されたmaterialを持つサブセットのメッシュを取得します。素材が指定されていない場合は、元の素材を使用したサブセットが返されます。

💡
ジオメトリックサブセットとは、モデルのジオメトリを抽出したものです。例えば、特定の条件を満たすすべてのIfcDoorsとIfcWindowsを含むサブセットを抽出し、それらをハイライトしたりエクスポートしたりすることができます。

関数の引数#

  • modelID IFC モデルの ID です。

  • material (オプション) サブセットに割り当てられた素材（ある場合）。

例#

const model = ifcModel.modelID;
const manager = loader.ifcLoader.ifcManager;
const subset = manager.getSubset(model);

createSubset#

IfcLoader.IfcManager.createSubset (
config: SubsetConfig
): object;

🧱✨ 新しい幾何学的サブセットを作成します。

関数の引数#

  • config 以下のオプションを持つ設定オブジェクトです。

    • scene モデルが配置されているシーン。

    • modelID モデルの ID です。

    • ids サブセットに適合するモデルのアイテムの ID を表す。

    • removePrevious このモデルの前のサブセットをこの素材で削除するかどうか。

    • material (オプション) サブセットに適用する素材。素材が指定されていない場合、サブセットには元の素材が適用されます。

    • customID (オプション) サブセットに付けるオプションのカスタム名。これにより、同じ材料で複数のサブセットを作成することができます。

例#

const model = ifcModel.modelID;
const manager = loader.ifcLoader.ifcManager;
const config = {
modelID: model
scene: scene,
ids: [id],
removePrevious: true
}
manager.createSubset(config);

removeFromSubset#

IfcLoader.IfcManager.removeFromSubset (
modelID: number,
ids: number[],
customID?: string,
material?: Material
): void;

🧱💥🤏 指定した幾何学的サブセットから指定した項目を削除します。

関数の引数#

  • modelID IFC モデルの ID です。

  • ids サブセットから削除するアイテムの ID。

  • customID (オプション) サブセットに付けるオプションのカスタム名。これにより、同じ材料で複数のサブセットを作成することができます。

  • material (オプション) サブセットに適用する素材。素材が指定されていない場合、サブセットには元の素材が適用されます。

import { IFCWALLSTANDARDCASE as W } from "web-ifc";
const model = ifcModel.modelID;

manager.removeFromSubset(model, walls);
