ハローワールド

はじめに#

⚡ IFC.js を使った BIM アプリケーションの作成はとても簡単です。ガイドの完全なプロジェクトこちらと、デプロイされたアプリケーションこちらをご覧いただけます。以下の手順を実行するには、Node.jsと任意の IDE がインストールされている必要があります。また、IFC ファイルをお持ちでない方は、こちらから入手できます。

💡
IFC.jsを使用するには、Web開発（HTML、CSS、JavaScript）とThree.jsの基本的な知識が必要です。Three.jsの使用経験がない場合は、こちらをご覧いただくとよいでしょう。

🐣 この最小限のチュートリアルは、React、Vue、Angular、Svelte などのフレームワークを使わずに、バニラ JavaScript で行います。しかし、同じ手順は、これらのユースケースのいずれにも適用することができます。

プロジェクトの立ち上げ#

ライブラリのインストール#

👩‍🏫 まず最初に、空のフォルダを作って、npm initというコマンドで新しい npm プロジェクトを立ち上げます。そうすると、プロジェクト名やバージョン、コマンド、依存関係などのデータが入ったpackage.jsonファイルが生成されます。さらに、以下の依存関係が npm とともにインストールされている必要があります。

// IFC.jsのインストール
npm i web-ifc-three
// Three.jsのインストール
npm i three
// バンドルラーのインストール：このガイドではrollup.jsを使用します。
npm i rollup --save-dev
npm i @rollup/plugin-node-resolve --save-dev

📝 次のステップでは、アプリケーションのメインドキュメントとして、index.htmlという名前の HTML ファイルを作成します。この HTML には以下が含まれます。

  • canvas 要素。Three.js のシーンのレンダリングに使用します。
  • input 要素。コンピュータからアプリケーションに IFC ファイルを開くために使用します。
  • script：bundle.jsというファイルを参照する。
<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8" />
<meta http-equiv="X-UA-Compatible" content="IE=edge" />
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
<link rel="stylesheet" href="styles.css" />
<title>Document</title>
</head>
<body>
<input type="file" name="load" id="file-input" />
<canvas id="three-canvas"></canvas>
<script src="bundle.js"></script>
</body>
</html>

スタイルの追加#

🎨 以下の CSS ファイルは、キャンバスをフルスクリーンにします。

* {
margin: 0;
padding: 0;
box-sizing: border-box;
}
html,
body {
overflow: hidden;
}
#three-canvas {
position: fixed;
top: 0;
left: 0;
outline: none;
}
#file-input {
z-index: 1;
position: absolute;
}

コードバンディング#

次に、ロールアップの設定ファイルを作成します。このファイルは、rollup.config.jsという名前で、先にインストールしたプラグインへの参照を含む必要があります。

🧻

Rollup?

Rollupは、非常に人気のあるバンドルライブラリです。例えば、Three.jsで使用されているバンドラーです。詳しくは、rollup docsをご覧ください。

import resolve from "@rollup/plugin-node-resolve";
export default {
input: "src/app.js",
output: [
{
format: "esm",
file: "src/bundle.js",
},
],
plugins: [resolve()],
};

🎯 また、package.jsonファイルを修正して、ロールアップを簡単に制御するためのコマンドを含める必要があります。各コマンドには、ロールアップ設定ファイルへの相対パスを指定する必要があります。すべてが正しくインストールされていれば、このファイルに同じ依存関係が見られるはずです（ライブラリのバージョンが異なる場合があります）。

  • npm run build は、プロジェクトをバンドルして、プロジェクトのルートディレクトリに bundle.js というファイルを作成します。

  • npm run watch は watch mode を有効にして、コードに変更を加えて保存するたびに、このファイルを自動的に更新します。

{
"name": "example",
"version": "1.0.0",
"description": "-",
"main": "app.js",
"scripts": {
"build": "rollup -c ./rollup.config.js",
"watch": "rollup -w -c ./rollup.config.js"
},
"author": "",
"license": "ISC",
"devDependencies": {
"@rollup/plugin-node-resolve": "^11.2.1",
"rollup": "^2.45.2"
},
"dependencies": {
"three": "^0.128.0",
"web-ifc-three": "0.0.102"
}
}

WebAssembly#

📁 次にやるべきことは、web-ifc.wasmとweb-ifc-mt.wasmをプロジェクト内のディレクトリにコピーすることです。web-ifc.wasmとweb-ifc-mt.wasmは、node_modules\web-ifc（Three's IFCLoader のみを使用している場合は、node_modulesThree\examples\jsm\loaders\ifc）の中にあります。今回の例では、プロジェクトのルートディレクトリにある wasm というフォルダにコピーします。

これらのファイルが必要なのは、IFC ファイルをネイティブな速度で読み書きするための解析コアであるweb-ifcのコンパイル済み C++ロジックが含まれているからです。

💡
これらのファイルは、アプリケーションの中で静的に提供されなければなりません。React、Angular、Vue、Svelteなどのフレームワークやライブラリを使用している場合は、異なる調整が必要になるかもしれません。

3D シーンの設定#

最後に，アプリケーションのコードを記述するための JavaScript ファイルを作成します。このファイルはどこにあっても、どんな名前でも構いませんが、rollup.config.jsに反映させる必要があります。

🧱 今回は、Three.js を使って、基本的な 3D シーンを作成します。

import {
AmbientLight,
AxesHelper,
DirectionalLight,
GridHelper,
PerspectiveCamera,
Scene,
WebGLRenderer,
} from "three";
import { OrbitControls } from "three/examples/jsm/controls/OrbitControls";
// Three.jsのシーンを作成します。
const scene = new Scene();
// ビューポートのサイズを格納するオブジェクト
const size = {
width: window.innerWidth,
height: window.innerHeight,
};
// カメラ（ユーザーの視点）の作成
const aspect = size.width / size.height;
const camera = new PerspectiveCamera(75, aspect);
camera.position.z = 15;
camera.position.y = 13;
camera.position.x = 8;
// シーンの照明を作成する
const lightColor = 0xffffff;
const ambientLight = new AmbientLight(lightColor, 0.5);
scene.add(ambientLight);
const directionalLight = new DirectionalLight(lightColor, 1);
directionalLight.position.set(0, 10, 0);
directionalLight.target.position.set(-5, 0, 0);
scene.add(directionalLight);
scene.add(directionalLight.target);
// レンダラを設定し、HTMLのキャンバスをフェッチします。
const threeCanvas = document.getElementById("three-canvas");
const renderer = new WebGLRenderer({
canvas: threeCanvas,
alpha: true,
});
renderer.setSize(size.width, size.height);
renderer.setPixelRatio(Math.min(window.devicePixelRatio, 2));
// シーンにグリッドと軸を作成する
const grid = new GridHelper(50, 30);
scene.add(grid);
const axes = new AxesHelper();
axes.material.depthTest = false;
axes.renderOrder = 1;
scene.add(axes);
// オービットコントロール（シーンをナビゲートするためのもの）の作成
const controls = new OrbitControls(camera, threeCanvas);
controls.enableDamping = true;
controls.target.set(-2, 0, 0);
// アニメーションループ
const animate = () => {
controls.update();
renderer.render(scene, camera);
requestAnimationFrame(animate);
};
animate();
// ビューポートをブラウザのサイズに合わせる
window.addEventListener("resize", () => {
size.width = window.innerWidth;
size.height = window.innerHeight;
camera.aspect = size.width / size.height;
camera.updateProjectionMatrix();
renderer.setSize(size.width, size.height);
});

🌏 アプリケーションをローカルで実行するには、ローカルサーバーが必要です。IDE として VS Code を使用している場合は、Live Server extensionをインストールするのも一つの方法です。これにより、Google Chrome のインスタンスを開き、ウェブアプリケーションを実行して、コードに加えた変更をリアルタイムで確認することができます。

IFC ファイルの読み込み#

ユーザーのモデルを読み込む#

🏠 最後に、IFC.js を使って IFC ファイルを読み込むことにします。これは、ローダーをインスタンス化して、ユーザーが IFC ファイルを HTML の入力要素にアップロードしたときのイベントを作成することで実現できます。

import { IFCLoader } from "web-ifc-three/IFCLoader";
// IFCの読み込みを設定する
const ifcLoader = new IFCLoader();
const input = document.getElementById("file-input");
input.addEventListener(
"change",
(changed) => {
const file = changed.target.files[0];
var ifcURL = URL.createObjectURL(file);
ifcLoader.load(ifcURL, (ifcModel) => scene.add(ifcModel));
},
false
);

🎣 プロジェクトのサーブファイルのルートに wasm ファイルを保存していない場合は、setWasmPathでその場所を指定する必要があることに注意してください。例えば、プロジェクトのルートにあるstaticというフォルダの中にあるwasmというフォルダに保存していた場合は、以下のようになります。

ifcLoader.ifcManager.setWasmPath("static/wasm/");

すべてが正しく行われていれば、ローカルサーバにthisのようなものが表示されているはずです。ここからは無限の可能性があります。

モデルの読み込み#

🤔 前のポイントでは、BIM モデルを直接読み込む方法を見ましたが、これは素晴らしいことです。しかし、ユーザーに BIM モデルをアップロードさせるのではなく、自分の BIM モデルを表示させたい場合はどうすればいいでしょうか？これはとても簡単です。一般的には 2 つの可能性があります。

  • 表示したいアプリケーション内に IFC がある場合。

  • 外部のストレージサービスから IFC を取得する必要がある。

    🎩 最初のケースでは、IFC ファイルの URL を参照するだけで十分です。つまり、アプリケーション内の相対パスです。例えば、IFC がプロジェクトのルートにある「models」というフォルダにある場合、アプリケーションの起動時にその IFC を次のようにして読み込むことができます。

ifcLoader.load("models/Example_model.ifc", (ifcModel) => scene.add(ifcModel));
💡
リモートストレージサービスからファイルを取得する方法は、使用するサービスによって異なります。しかし、情報を取得してURLを作成し、それをIFCLoaderの引数として渡すというロジックは同じです。

IFC.jsでは他にどんなことができますか？
