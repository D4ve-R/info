30分でCDEを構築

2021年9月27日 · 1分間の読書

Ansoni

自分だけのCDEを一から作ってみよう

大手企業がこぞって CDE を販売しています。あなたも自分の CDE を作ってみませんか？

ソース

しかし、CDE とは何でしょうか？#

📦 非公式に言えば、CDEとは、建設プロジェクトの情報が一元化され、インターネットでアクセスできる場所にすぎない。

{" "}

🔧🔨 CDEは、従来のBIMアプリケーションのすべての機能を持つことができます。3Dビューア、データの読み込みと編集、エネルギーシミュレーション、自動計測と予算管理などです。

️

🌎🌍🌏 しかし、これらの製品には基本的な利点があります。それは、ウェブ上にあるということであり、どこからでも、どんなデバイスからでも、何もインストールすることなくアクセスできるということです。

分かったが、なぜ CDE が必要なのか？#

💸 しかし、実は今、CDE はニューブラックなのです。**業界のほとんどの大手企業が CDE を構築し、法外な価格で顧客に販売しています。

🏃‍♀️🏃‍♂️🏃‍♀️ インターネットの高速化・高機能化に伴い、私たちが知っているようなデスクトップアプリケーションが通用しなくなり、私たちの仕事の大部分がウェブアプリケーションやサーバーアプリケーションで行われるようになる日が来るでしょう。言い換えれば、CDE は未来であり、現在、最高の CDE を作るための大きな競争が行われています。

CDE の素晴らしさはわかった。でも、自分では絶対に開発できません。#

👩‍🏫 間違っている！？BIM ソリューションができないと思われている最大の原因は、現在の BIM ベンダーにあると思います。IFC.js では、この神話を打ち破り、シンプルな CDE を一から作り、BIM アプリケーションの開発を始めることはそれほど難しいことではないことを示したいと考えました。

☝ 明らかに、このチュートリアルは最小限の例です。しかし、このチュートリアルのシンプルな CDE から大企業の CDE までは、見かけほどの距離はありません。雑談はここまでにして、さっそくやってみましょう！。

CDE デザイン#

戦略#

⏳ この CDE を行うための時間は30 分しかないので、時間を無駄にしないために、慎重に方法を選択しなければなりません。まず第一に、プロジェクトのドキュメントを保存する場所が必要です。それを可能にする既存のツールをご存知でしょうか？私たちは、Google Drive**を考えました。

{" "}

💪 Google Driveには、ファイルの読み書きのコントロール、バージョン管理、オープンAPIなど、多くの機能があります。要するに、BIMソフトウェア会社が有料で提供しているものの多くを、完全に無料で提供しているのです。

️

🛑 しかし、Google Drive には、3D ビューアがなく、BIM モデル（IFC モデルなど）のデータに簡単にアクセスできないという根本的な限界があります。

🤝 ここで IFC.js の出番です。、ウェブ上で直接 BIM アプリケーションを構築することができます。Google Drive のストレージとアクセス権を IFC.js の 3D とデータの読み取りと編集機能と組み合わせることで、基本的な CDE を 30 分で構築することができました。その方法をご紹介します。

建築#

🤷‍♀️ IFC.js と google drive を連携させ、CDE として使用するにはどうすればよいでしょうか？私たちのデザイン案は以下の通りです。複雑に見えるかもしれませんが、実はとてもとてもシンプルです。

🏠 まず最初に、google driveからファイルをダウンロードできる3D BIM ウェブアプリケーションを作成してみます。具体的には、URL パラメータで見つかった ID のファイルを google drive からダウンロードします。例えば、URL がhttps://.../?id=123&name=TEST.ifcであれば、アプリにログインしているユーザーの google アカウントからIDが123のファイルをダウンロードします。

🌐 次に、3 つのことを行うgoogle chrome extensionを作成します。

  • ウェブスクレイピングでユーザーがクリックした HTML 要素の名前を読み取る。

  • google drive API を使用して、その要素の ID を調べます。

  • IFC.js の Web アプリケーションで URL を開くことができるメニューを作成します。さらに、ファイルの名前と ID という 2 つの URL パラメータを追加することになります。

    {" "}

    要するに、google chromeの拡張機能は、ユーザーがクリックするとファイルの名前を取得し、そのIDでファイルをダウンロードする方法を知っているウェブアプリを開きます。以上です。

    ️

IFC.js アプリ#

BIM アプリ#

🏢 IFC.js のおかげで、ここが一番簡単です。具体的には、50 行のコードで BIM アプリケーションを作ることができるweb-ifc-viewerを使って作成してみましょう。すべてのコードはここで見ることができます。

😉 はい、その通りです。50 行のフル 3D BIM アプリです。しかも、断面図や 3D 寸法などの機能も備えています。

import {IfcViewerAPI} from 'web-ifc-viewer';
import {Color} from 'three'
// 3Dシーンがレンダリングされるdivコンテナの取得
const container = document.getElementById('viewer-container');
// IFC.jsのAPIを初期化し、グローバル変数として追加する。
const viewer = new IfcViewerAPI({
container,
backgroundColor: new Color(0xffffff)
});
viewer.IFC.applyWebIfcConfig({
COORDINATE_TO_ORIGIN: true,
USE_FAST_BOOLS: true
});
window.webIfcAPI = viewer;
// シーンの設定
viewer.addAxes();
viewer.addGrid(50, 50);
viewer.IFC.setWasmPath('wasm/');
viewer.clipper.active = true;
let dimensionsActive = false;
// 基本的な入力ロジックの追加
const handleKeyDown = (event) => {
if (event.code === 'KeyE') {
dimensionsActive = !dimensionsActive;
viewer.dimensions.active = dimensionsActive;
viewer.dimensions.previewActive = dimensionsActive;
viewer.IFC.unPrepickIfcItems();
window.onmousemove = dimensionsActive ?
null :
.IFC.prePickIfcItem;
}
if (event.code === 'KeyD') {
viewer.dimensions.create();
}
if (event.code === 'KeyG') {
viewer.clipper.createPlane();
}
if (event.code === 'Delete') {
viewer.dimensions.deleteAll();
viewer.clipper.deletePlane();
viewer.IFC.unpickIfcItems();
}
};
window.onkeydown = handleKeyDown;
// アイテムにカーソルを合わせるとハイライトされる
window.onmousemove = viewer.IFC.prePickIfcItem;
// アイテムの選択とログのプロパティ
window.ondblclick = async () => {
const item = await viewer.IFC.pickIfcItem(true);
if(item.modelID === undefined || item.id === undefined ) return;
console.log(
await viewer.IFC.getProperties(item.modelID, item.id, true));
}

Google Drive API#

🔒🔒🔒 さて、BIM ビューアですが、今のところGoogle Drive APIからは何もダウンロードできません。公式ドキュメント](https://developers.google.com/drive/api/v3/quickstart/js)に記載されている手順に従い、JavaScriptのコードをコピー＆ペーストするだけで、APIにアクセスできるようになります。

{" "}

🔑🔑🔑 Googleのガイドに従って、新しいGoogle Cloudプロジェクトを作成し、自分のCLIENT_IDとAPI_KEYを取得する必要があります。これは、Googleがあなたのことを知るために必要な2つのテキストです。言い換えれば、私たちが持っているキーを使ってあなたのドライブにアクセスすることはできません。

️

📋 私たちが投稿したコードを見てみると、私たちはgoogle exampleからほとんどすべてのコードをコピー＆ペーストしていることがわかります。追加したのは以下の部分だけで、これは単純にウェブアプリケーションの URL からファイルの ID を取得し、google drive からダウンロードしています。

/**
* サインインした状態が変化したときに呼び出され、UIを適切に更新します。サインイン後は、APIが呼び出されます。
*/
function updateSigninStatus(isSignedIn) {
if (isSignedIn) {
const { id, name } = getIdAndName();
fileName.innerText = `Name: ${name}`;
if (id) downloadFile(id);
else console.log("No ID was providen as URL parameter!");
}
}
/**
* ボタンをクリックすると、ユーザーがサインインします。
*/
function handleAuthClick(event) {
authorizeButton.onclick = handleSignoutClick;
buttonText.innerText = "Sign out";
gapi.auth2.getAuthInstance().signIn();
}
function downloadFile(fileId) {
gapi.client.drive.files
.get({
fileId: fileId,
alt: "media",
})
.then(
function (response) {
loadIfcModel(response.body);
},
function (error) {
console.error(error);
}
);
}
function getIdAndName() {
const urlParams = new URLSearchParams(window.location.search);
const id = urlParams.get("id");
const name = urlParams.get("name");
return { id, name };
}
function loadIfcModel(text) {
const blob = new Blob([text], { type: "text/plain" });
const file = new File([blob], "ifcModel");
window.webIfcAPI.IFC.loadIfc(file);
}

🎉🎉 おめでとうございます。これで、Google Drive からファイルをダウンロードできる BIM アプリケーションが完成しました。あとは、最後の仕上げとして、Google Chrome の拡張機能を作成するだけです。

さて、これから Google Chrome の拡張機能を作って、ユーザーが開きたいファイルを選択できるようにします。今までやったことがない人には難しく聞こえるかもしれませんが、実際にはとても簡単です。完成したものはこちらでご覧いただけます。

️

👣 インターネット上にはたくさんのstep-by-step guideがありますが、私たちは、私たちのリポジトリからコードを取り出し、それを研究し、それがどのように機能するかを遊んでみることをお勧めします。

🧱 重要なファイルは 2 つしかありません。

  • background.js: ユーザーが何かをクリックしたときなど、ページのイベントを制御しますが、ページのコンテンツにはアクセスできません。

  • foreground.js: ページのコンテンツにはアクセスできますが、イベントにはアクセスできません。

👬 想像できるように、私たちが望むことを実行するためには、両方のファイルがお互いに通信しなければなりません。コードを見るとわかるように、シンプルなメッセージシステムでそれを行っています。

👩‍🏫 リポジトリには、何が何をするのかについてのステップバイステップのコメントがあります。関連するコードは全部で 100 行にも満たず、他はすべてコピー＆ペーストです。

みんなで一緒に!#

結論#

🏆 おめでとうございます。あなたは最初の CDEを作成しました。大したことではないと思われるかもしれませんが、その機能性を考えてみてください。ファイルの保存、パーミッションとバージョンコントロール、3D ビュー、プロパティへのアクセス、3D 測定、断面図...。

🚀🚀🚀 Google Drive を使って短時間でこれだけのことができたのですから、IFC.js や、データベースやサーバーなどの他の技術を使って何ができるか想像できますか？ご心配なく。
